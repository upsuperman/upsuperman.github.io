hexo+github搭建博客

Posted on 2019-03-17
Words count in article: 584 | Reading time ≈ 2

参考各家搭建博客的心得，千呼万唤始出来，搭好了自己的博客，简单记录一下使用hexo+github搭建的过程。hexo是一个基于Node.js的静态博客框架，我们的博客在本地生成，然后托管到github上。每次编辑markdown格式的文章，然后发布出去。
参考GitHub+Hexo 搭建个人网站详细教程

一.准备

  1. 进入github，创建新的repository，注意repo的名字必须为github的用户名+github.io，比如我的upsuperman.github.io。
  2. 安装git，进行本地与github的非对称加密。

  1. Yilia主题, 主要修改blog/_config.yml配置文件和blog/theam/_config.yml主题配置文件，关于模块更改参考hexo yilia主题使用方案
  2. Next主题，关于主题设置和使用参考上述github链接，不再赘述。

四. 发布文章

使用hexo new "文章"新建文章，然后按照markdown语法编辑，需要显示部分内容，在正文想显示的部分加入<!-- more -->，注意tags:后面需要加空格。

五. 使用技巧说明

  1. 使用markdownpad插入图片

    • 更改主页配置文件_config.yml 里的post_asset_folder: 将选项设置为true；
    • hexo目录下执行npm install hexo-asset-image –save，下载安装一个可以上传本地图片的插件；
    • 运行hexo n “xx”来生成md博文时，/source/_posts文件夹内除了xx.md文件还有一个同名的文件夹
    • 在xx.md中想引入图片时，先把图片复制到xx这个文件夹中，然后只需要在xx.md中按照markdown的格式引入图片：(xx/图片名.jpg)

  2. 使用markdownpad插入表格

    • 更改markdownpad的选型，将Tools->Options->Markdown processor的markdown更改为markdown(extral)，扩展版本才支持表格的语法。
